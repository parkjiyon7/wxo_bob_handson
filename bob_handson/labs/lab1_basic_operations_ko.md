**소요 시간:** 30분
**난이도:** 초급
**사전 요구사항:** Bob 설치 및 실행 중

## 🎯 학습 목표

이 실습을 완료하면 다음을 수행할 수 있습니다:
- 코드베이스에서 파일 탐색 및 읽기
- 특정 코드 패턴 검색
- Bob을 사용하여 간단한 코드 변경하기
- 명령 실행 및 결과 확인

- Bob 모드 간 전환 (Code, Ask, Plan)
- 코드 개선을 위한 Bob Findings 활용
- 신뢰할 수 있는 작업을 위한 자동 승인 구성 및 사용

### Exercise 1: 파일 탐색 및 읽기 (5분)

2. 메인 설정 파일 읽기
3. 애플리케이션의 진입점 식별

**사용할 Bob 도구:**

**예시 프롬프트:**

**💡 실시간 가치 지표:**
Bob이 파일 내용을 몇 초 만에 읽고 표시하는 것을 확인하세요. 여러 디렉토리를 수동으로 탐색하고 파일을 여는 것과 비교하면 탐색 작업당 약 2-3분을 절약할 수 있습니다.

**예시 프롬프트:**

### Exercise 9: 자동 승인 - 신뢰할 수 있는 작업 간소화 (5분)

**시나리오:** 신뢰할 수 있는 작업의 경우 자동 승인을 구성하여 워크플로우를 가속화할 수 있습니다.

**자동 승인이란?**
- 특정 Bob 작업을 자동으로 승인
- 안전한 작업에 대한 왕복 감소
- 신뢰 수준에 따라 구성 가능
- 작업 유형별로 활성화/비활성화 가능

**자동 승인을 사용해야 하는 경우:**
- 파일 읽기 (항상 안전)
- 코드 검색 (수정 없음)
- 파일 및 디렉토리 나열
- 읽기 전용 명령 실행

**자동 승인을 사용하지 말아야 하는 경우:**
- 중요한 파일에 쓰기
- 파괴적인 명령 실행
- 데이터베이스 변경
- 프로덕션 배포

**작업:**
1. 현재 자동 승인 설정 이해
2. 읽기 작업에 대한 자동 승인 구성
3. 자동 승인이 활성화된 워크플로우 테스트

**예시 구성:**
```
자동 승인 활성화:
- read_file
- list_files
- search_files
- list_code_definition_names

수동 승인 필요:
- write_to_file
- apply_diff
- execute_command
- insert_content
```

**기대 결과:**
- 자동 승인의 이점과 위험 이해
- 자동 승인하기에 안전한 작업 알기
- 설정을 적절하게 구성
- 신뢰할 수 있는 작업에 대한 더 빠른 워크플로우

**핵심 학습:**
- 자동 승인은 안전한 작업을 가속화
- 수정 사항에 대해서는 항상 승인 필요
- 편안한 수준에 따라 구성
- 속도와 안전의 균형

## 🎓 핵심 요점

이 실습을 완료한 후 다음을 이해해야 합니다:

1. **파일 작업**
   - 탐색을 위한 `list_files`
   - 내용 보기를 위한 `read_file`
   - 수정하기 전에 항상 읽기

2. **검색 및 발견**
   - 패턴 매칭을 위한 `search_files`
   - 코드 구조를 위한 `list_code_definition_names`
   - 강력한 검색을 위해 정규식 사용

3. **코드 편집**
   - 정확한 변경을 위한 `apply_diff` (권장)
   - 새 파일 또는 전체 재작성을 위한 `write_to_file`
   - 줄 추가를 위한 `insert_content`
   - 항상 Bob에게 컨텍스트 제공

4. **명령 실행**
   - CLI 작업을 위한 `execute_command`
   - 테스트를 실행하여 변경 사항 확인
   - 명령을 사용하여 정보 수집

5. **리터럴 코딩**
   - Bob은 코드를 자연어로 설명
   - 익숙하지 않은 코드베이스를 이해하는 데 Bob 사용
   - 코드 품질 및 개선에 대한 통찰력 얻기
   - 학습 및 온보딩 가속화

6. **Bob 모드**
   - **Ask 모드**: 질문 및 설명
   - **Plan 모드**: 전략 및 작업 분해
   - **Code 모드**: 구현 및 변경
   - 필요에 따라 모드 전환

7. **Bob Findings**
   - 자동 보안 취약점 감지
   - 코드 품질 분석
   - 성능 문제 식별
   - 사전 개선 제안

8. **자동 승인**
   - 신뢰할 수 있는 작업 가속화
   - 읽기 전용 작업에 대해 구성
   - 수정 사항에 대해서는 항상 승인 필요
   - 속도와 안전의 균형

9. **모범 사례**
   - 프롬프트에서 구체적으로 작성
   - 달성하려는 것에 대한 컨텍스트 제공
   - 복잡한 작업을 단계로 분해
   - 변경 후 확인
   - 작업에 적합한 모드 사용

## 💡 성공을 위한 팁

1. **간단하게 시작:** 변경하기 전에 읽기 작업부터 시작
2. **구체적으로 작성:** 명확한 프롬프트가 더 나은 결과를 얻음
3. **컨텍스트 제공:** Bob에게 달성하려는 것을 알려주기
4. **변경 사항 확인:** 변경 사항이 예상대로 작동하는지 항상 확인
5. **질문하기:** 명확하지 않은 것이 있으면 Bob에게 설명 요청

## 🐛 일반적인 문제

### 문제: Bob이 파일을 찾을 수 없음
**해결책:** 먼저 `list_files`를 사용하여 올바른 경로 확인

### 문제: 변경 사항이 올바르게 적용되지 않음
**해결책:** `read_file`을 사용하여 현재 상태를 확인한 다음 정확한 내용으로 다시 시도

### 문제: 명령 실패
**해결책:** 오류 메시지를 확인하고 명령 구문 조정

**해결책:** Bob에게 물어보세요! "[작업을 수행]하는 가장 좋은 방법은 무엇인가요?"

## 💰 비즈니스 영향

이 실습은 즉각적인 생산성 향상을 제공하는 Bob의 핵심 기능을 보여줍니다:

### ⏱️ 생산성 향상
- **파일 탐색**: 수동 탐색보다 60% 빠름 (작업당 2-3분 절약)
- **코드 검색**: 대규모 코드베이스에서 grep/IDE 검색보다 75% 빠름
- **코드 변경**: 일상적인 편집 및 리팩토링 시간 50% 단축
- **명령 실행**: IDE와 터미널 간 컨텍스트 전환 제거

### 🐛 품질 개선
- **Bob Findings**: 일반적인 코드 품질 문제의 85%를 자동으로 감지
- **보안 스캔**: 코드 검토 전에 취약점 식별
- **일관성**: 여러 파일에서 균일한 코드 변경 보장
- **오류 방지**: 적용하기 전에 변경 사항 검증

### 🔒 보안 강화
- **자동 보안 분석**: SQL 인젝션, XSS 및 기타 취약점 포착
- **모범 사례 적용**: 보안 안티 패턴 식별
- **종속성 스캔**: 오래되거나 취약한 패키지 플래그 지정

### 💼 효율성 영향
- **수동 작업 감소**: 개발자당 주당 4-6시간 절약
- **더 빠른 온보딩**: 신규 개발자가 40% 더 빠르게 생산성 향상
- **프로덕션 문제 감소**: 일상적인 변경으로 인한 버그 70% 감소

**개발자당 주간 예상 가치**: 생산성 향상으로 4-6시간 절약

## 🌟 이 실습의 Bob 차별화 요소

이러한 실습을 통해 Bob의 고유한 기능을 경험했습니다:

### 1. **리터럴 코딩** (Exercise 6)
기존 AI 어시스턴트와 달리 Bob은 명령을 실행하는 것뿐만 아니라 코드를 자연어로 이해하고 설명합니다. 이는 온보딩을 가속화하고 팀이 복잡한 코드베이스를 더 빠르게 이해하도록 돕습니다.

### 2. **다중 모드 인텔리전스** (Exercise 7)
Bob의 Ask, Plan 및 Code 모드는 다양한 작업에 최적화됩니다. 이것은 단순한 UI가 아닙니다. 각 모드는 더 나은 결과를 위해 다른 추론 전략과 모델 선택을 사용합니다.

### 3. **Bob Findings** (Exercise 8)
사전 보안 및 품질 분석이 자동으로 실행됩니다. Bob은 요청을 기다리지 않고 작업하면서 문제를 식별하여 프로덕션에 도달하기 전에 문제를 포착합니다.

### 4. **스마트 자동 승인** (Exercise 9)
구성 가능한 신뢰 수준을 통해 속도와 안전의 균형을 맞출 수 있습니다. Bob은 사용자의 선호도를 학습하고 중요한 변경 사항에 대한 제어를 유지하면서 신뢰할 수 있는 작업을 간소화합니다.

### 5. **컨텍스트 인식 작업**
Bob은 여러 파일을 함께 읽고, 코드 간의 관계를 이해하며, 복잡한 워크플로우에서 컨텍스트를 유지합니다. 이는 실제 개발 작업에 필수적입니다.
